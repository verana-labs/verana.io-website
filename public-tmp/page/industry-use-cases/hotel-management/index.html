The Problem: Hotel Discovery & Broker Domination

The hotel industry is a striking example of how centralization locks value. Today, nearly all hotel reservations happen through brokering platforms like Booking.com, Expedia, Agoda, or Airbnb. These intermediaries capture a huge share of hotel revenues and quietly raise room costs for travelers.

The Paradox

While users believe they’re saving money, they actually pay more, since hotels must increase their prices to cover broker fees.

Market Reality

If a hotel isn’t listed on broker platforms, it effectively doesn’t exist in the market.

Current Centralized Model

Travelers

Search and book through centralized platforms

Brokers

Control visibility and extract high commissions

Hotels

Pay high commissions for visibility

Hotel Management Systems Today

To manage their properties, hotels rely on Hotel Management Systems (HMS)—either proprietary SaaS or open source solutions. Open source HMS let properties keep control, but discovery is still broker-controlled.

Why Open Source HMS Appeals to Hotels

No Vendor Lock-in

Host software anywhere and maintain full control.

Data Ownership

Hotels retain complete ownership of their data.

Key Open Source Modules

  • Property Management System (PMS)
  • Booking Engine
  • Hotel Website Builder
  • Hotel Mobile App Skeleton

Software Add-ons

Channel managers that sync inventory with broker platforms.

Custom Development

Tailored features for specific hotel groups or specialist operators.

Managed Hosting

Running the HMS stack for properties that don’t want to self-host.

The Visibility Problem Remains

These systems work well for hotel operations, but visibility is still dictated by brokers. Even when hotels run their own sites or apps, most new guests still arrive via the broker monopoly.

Enter Verana: Making Hotels Instantly Discoverable

Verana changes the game by removing dependency on centralized brokers.

1

Create a Hotel Ecosystem in Verana

  • Establish rules via an Ecosystem Governance Framework (EGF).
  • Create a Trust Registry and define a Hotel Credential Schema in Verana.
2

Add Verifiable Trust to Open Source HMS Software

Integrate Verana’s Verifiable Trust stack into booking engines, PMS, and mobile apps.

3

Launch a Verifiable User Agent

  • Build a global mobile app or website (the “HMS Provider App”).
  • Function as a browser for verifiable hotels.
  • Act as a broker competitor—without extracting rents.
