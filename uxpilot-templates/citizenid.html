Implementation Guide Credential Schemas Governance Framework

Verana provides governments with full open source software and infrastructure to issue privacy-preserving, verifiable, decentralized Digital IDs that empower citizens while maintaining national sovereignty over identity systems.

Create the Digital Identity Ecosystem

Governments establish:

Ecosystem Governance Framework (EGF)

Sets rules for Citizen ID issuance, verification, and authorization of participants, including pay-per-verification business model if desired.

Ecosystem Trust Registry

Create in Verana to match the EGF, including the Citizen ID Credential Schema.

Onboard Ecosystem Participants

Start onboarding participants as explained in the EGF, using Verana's trust registry process. Participants can be verifiers or trust registry operators.

Build a Digital Wallet

Governments build a Digital Wallet mobile app that acts as a Verifiable User Agent (VUA).

Store Credentials

Citizens use the wallet to store their Citizen ID Verifiable Credential (VC)

Self-Sovereign

Citizens own their credentials, with no central account or dependency

Open Standards

Compatible with EIDAS2, DIDs, DIDComm, W3C VCs, OpenID4VC, OpenID4VP

Citizen Onboarding into the Central Database

At Public Offices (e.g., civil registry):

1

Citizen biometric data is captured and documents validated

2

Citizen data is recorded in the central registry database

3

Citizen receives an NFC-enabled ID card containing personal and biometric information

This ensures official enrollment into the state identity system.

Issuing the Citizen ID Verifiable Credential

Citizens obtain their Citizen ID Verifiable Credential by downloading the Government Digital Wallet and completing a secure identity validation process.

Example Enrollment Flow:

1

Citizen captures photo of their physical ID card. App reads the MRZ to retrieve NFC access key

2

Citizen enters the PIN code of the card to authorize the process

3

Citizen taps ID card on handset, enabling NFC read to extract personal and biometric information

4

App performs live face match against extracted ID photo to confirm legitimate cardholder

5

If all checks succeed, a Citizen ID Verifiable Credential is securely issued to the digital wallet

Authorized Verifiers

The government decides who can request credential presentations:

Validation Process

Applicants (e.g., banks, telecoms, hospitals) undergo a validation process before being authorized.

Authorization

Once approved, they are registered as authorized verifiers for the Citizen ID schema.

Pay-per-Verification Model

Each verification request incurs a fee

Fee payment is privacy-preserving

Issuers cannot know which citizen presented credentials to verifiers

Selective Disclosure

By supporting advanced credential formats (e.g., ZKPs, BBS+, Anoncreds), governments can enable:

Minimal Disclosure

Citizens can prove attributes without revealing full data (e.g., "over 18" without disclosing date of birth)

Privacy Protection

Minimal disclosure principles reduce risks of data misuse
