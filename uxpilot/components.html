Overview of Verana Components

Verana's architecture is built on modular, open-source components that work together to create a decentralized trust infrastructure. Each component serves a specific purpose and can be deployed independently or as part of the complete Verana ecosystem.

Component Architecture

Component Spotlight

Network Components

Verana Node

Cosmos SDK Layer-1
Testnet Repository

A layer-1 Cosmos SDK based implementation of the verifiable public registry spec. Verana is a registry of trust registries, where any ecosystem can create and self-govern its trust network.

Governance of the chain is managed by a council.

Indexer

Horoscope v2 Fork Database
Testnet Repository

A fork of Horoscope v2, the indexer of the Aura Network. The purpose of the Verana indexer is to build a local database by executing the transaction of each block of the verana network from first block, to be able to:

  • Perform advanced queries that cannot be done with the ledger API
  • Maintain state change history, to get access to information that is not visible in current ledger state

Resolver

Trust Engine Query API
Testnet Repository

The purpose of the Verana resolver is to build and maintain up to date a local database, by querying the indexer and subscribe to change events of the DID Directory, to be able to:

  • Crawl verifiable services, dereferences their credential, and organize data
  • Provide a universal query API to Verifiable User Agents and to search engine, for searching verifiable data

For a better understanding, you should learn about the trust engine

Verifiable Service Components

Note: Verana is DID method agnostic. You can use any DID method for your verifiable services. However, make sure that the trust resolver knows how to resolve your favorite DID method (add the plugin of your favorite DID method if not already available), else your verifiable services won't be resolvable, their verifiable data won't be indexed, and thus they would not appear in the search results.

Here is the list of open source software that you can use to build your verifiable services.

Credo Verifiable Service Agent

Credo-TS OpenWallet Agent
Repository

An agent for building Verifiable Services, based on OpenWallet Foundation credo-ts:

Features
  • Use as a container or npm package
  • Issue and verify credentials
  • Publish credentials to your DID Document
  • Supports DIDComm and OpenID4VC / OpenID4VP
Supported Standards
DID Methods:
did:peer did:web did:webvh did:cheqd
Credential Types:
W3C v1.1 AnonCreds
